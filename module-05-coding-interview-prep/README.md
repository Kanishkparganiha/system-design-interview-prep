## How to Use

1. Open a topic notebook
2. Read the problem description carefully
3. Write your solution in the starter code cell
4. Run the test cell — passing asserts means your solution is correct
5. If stuck, peek at the **Hints** cell at the bottom of each problem
6. Ask for solutions only when you want to review after solving

## Tips

- **Solve first, optimize later.** Get a brute force working before thinking about optimal.
- **Think out loud (even when solo).** Narrating your approach trains you for real interviews.
- **Know your complexity.** After solving, always ask: what is the time and space complexity?
- **Revisit failures.** If a test fails, read the error, don't just change random things.
- **Spaced repetition.** Come back to topics you found hard after a few days.
